# Review Discovery Engine — Vercel / Next.js Dashboard

A **separate**, static Next.js (App Router) front end that replicates the
Streamlit dashboard using the **frozen** data only. It does not scrape,
reclassify, or modify anything — it reads three JSON files exported from the
frozen dataset + precomputed engine output.

> The Streamlit/Render app (`dashboard/app.py`) remains the live Python version
> and is untouched. This folder is an additive Vercel deployment layer.

## Pages

1. **Overview** — KPIs, category mix, sentiment, monthly trend
2. **Reviews** — filterable Review Explorer (balanced sample; real totals shown)
3. **Themes** — discovery-problem & rec-complaint clusters, unmet needs, emerging themes
4. **Segments** — platform/region/language/user-type tables + most-affected
5. **PM Priority Radar** — Impact × Frequency scatter + ranked table + insight cards
6. **Weekly Pulse** — period-over-period KPIs, rising issues, notable discoveries

If a metric is missing from the exports, the UI shows **"Not available"**.

## Data

Static JSON in `public/data/` (generated by `scripts/export_vercel_data.py`):
- `engine_output.json` — precomputed metrics/insights/segments/radar/pulse
- `dashboard_summary.json` — **real counts from the full 26,823-row dataset**
- `reviews_sample.json` — balanced sample for the Explorer (full total preserved)

No Python runs at runtime; the app is fully static (`output: "export"`).

## Local run

```bash
# 1. (re)generate the frozen JSON exports from the repo root:
python3 scripts/export_vercel_data.py

# 2. build + run the Next.js app:
cd vercel-dashboard
npm install
npm run dev        # http://localhost:3000   (or: npm run build → static ./out)
```

## Deploy to Vercel

1. Push the repo to GitHub.
2. Vercel → **New Project** → import the repo.
3. Set **Root Directory** = `vercel-dashboard`.
4. Framework preset auto-detects **Next.js**; build command `next build`,
   output is static (`output: "export"`). Click **Deploy**.

To refresh data later: re-run the export script, commit the updated
`public/data/*.json`, and Vercel redeploys. The frozen dataset stays the source
of truth.
